# AutoSaving Alarm 原生 Windows 小工具方案

## Summary
基于 `C# + WinForms` 做一个原生 Windows 托盘常驻工具，核心目标是“每隔 N 分钟固定提醒一次你去保存”。  
交互定为：托盘常驻、正常状态绿色图标、到点切换为醒目色；同时弹出通知和右下角置顶小窗；首次启动先配置，之后记住配置，并提供“是否开机自启动”开关。

## Key Changes
- 应用形态
  - 使用 `ApplicationContext + NotifyIcon` 做无主窗口托盘应用，默认驻留系统托盘。
  - 首次启动若没有配置，直接打开设置窗口；后续按已保存配置启动。
  - 提供托盘菜单：`立即开始/暂停提醒`、`我刚保存了`、`设置`、`开机自启动`、`退出`。

- 提醒调度
  - 提醒模式采用“固定周期提醒”，以“开始计时/应用新配置”的时刻作为周期锚点。
  - 例如 `N=15` 时，按固定节奏触发；用户点击“已保存”只关闭本次提醒并记录时间，不重置下一次提醒周期。
  - 修改 `N` 或手动重新开始时，重新建立新的固定周期锚点。
  - 调度实现不依赖链式延时，改用“当前时间 vs 下一触发时间”计算，减少漂移，便于处理系统休眠恢复。

- 提醒表现
  - 平时只显示托盘图标，不显示常驻桌面窗。
  - 到点后触发两类提醒：
    - 托盘通知区气泡提醒
    - 右下角置顶小窗，显示“该保存了”“距离下次提醒还有多久”等状态
  - 小窗只在提醒时出现；点击“已保存”后隐藏。
  - 本版不加声音，避免打扰；后续可扩展提示音。

- 配置与持久化
  - 配置存到当前用户目录，例如 `%AppData%\\AutoSavingAlarm\\settings.json`。
  - 配置模型建议包含：
    - `interval_minutes`
    - `auto_start_enabled`
    - `start_with_windows`
    - `anchor_time`
    - `is_paused`
    - `last_acknowledged_at`
  - 开机自启动优先使用当前用户 `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run`，不要求管理员权限。
  - 首次使用默认手动启动；设置中可切换“以后开机自启动”。

- 主要模块
  - `TrayAppContext`：托盘图标、菜单、生命周期管理
  - `ReminderScheduler`：固定周期计算、暂停/恢复、休眠恢复补偿
  - `SettingsStore`：JSON 配置读写
  - `SettingsForm`：N 分钟、自启动、启停控制
  - `ReminderWindow`：置顶小窗、确认关闭当前提醒

## Public Interfaces / User-Facing Contract
- 用户可配置项
  - `提醒间隔 N 分钟`
  - `是否开机自启动`
  - `是否暂停提醒`
- 用户操作语义
  - `我刚保存了`：仅结束当前提醒展示，不改变固定周期
  - `暂停提醒`：停止后续提醒，直到用户手动恢复
  - `应用新间隔`：立即以当前时刻重建提醒周期
- 托盘状态约定
  - 绿色：正常计时中
  - 醒目色：当前处于“该保存了”的提醒态
  - 灰色：已暂停

## Test Plan
- 首次启动无配置时，自动进入设置界面；保存后正常进入托盘模式。
- 设置 `N=1` 分钟，确认每到 1 分钟固定触发提醒。
- 点击“已保存”后，本次提醒关闭，但下一次仍按固定周期到达，而不是重置为新的 1 分钟。
- 修改提醒间隔后，新的提醒节奏从修改时刻重新计算。
- 托盘图标在“正常/提醒/暂停”三种状态之间正确切换。
- 切换“开机自启动”时，注册表启动项正确写入和清除。
- 系统休眠超过一个周期后恢复，应用应立即进入提醒态，并继续推算下一次周期。
- 关闭设置窗口或提醒小窗时，不退出托盘进程；只有点“退出”才真正结束程序。
- 退出后重开，能恢复上次配置和状态。

## Assumptions
- v1 只做“提醒你保存”，不尝试监听其他软件的真实保存动作。
- v1 不做全局快捷键、不做多套配置、不做提示音、不做云同步。
- 交付目标默认是一个可发布的 Windows 单文件桌面程序，运行环境以当前机器可用的 `.NET WindowsDesktop` 为准。
- “Win 原生 API”在本方案中解释为“采用 Windows 原生桌面栈和系统能力”，即 `WinForms + NotifyIcon + 注册表自启动 + 原生窗口行为`，不下沉到纯 Win32/C++。
